Urgency Calculation

Relevant source files
  • README.md
  • tklr/controller.py

This page documents tklr's urgency calculation system, which determines task prioritization using a TaskWarrior-inspired scoring methodology. The urgency system evaluates multiple factors to assign numerical scores that influence how tasks are ordered in views and presented to users.

For information about how tasks are displayed and ordered using urgency scores, see User Interface. For details about task status management that feeds into urgency calculations, see Status Management.

Overview

The urgency calculation system applies weighted scoring across multiple dimensions to rank tasks by priority. The system evaluates factors including due dates, task age, importance levels, blocking relationships, and various metadata attributes. Each component contributes a weighted score that combines into a final urgency value used for task ordering.

Due Date Urgency Calculation

The due date component uses a time-based sliding scale that provides maximum urgency for overdue tasks and decreasing urgency for future due dates:

The urgency_due() function implements the following algorithm:

  • Overdue (≥7 days past): Returns 1.0 coefficient (maximum 16.0 points)
  • Near-term (-14 to +7 days): Linear scale from 0.2 to 1.0 coefficient
  • Future (>14 days ahead): Returns 0.2 coefficient (minimum 3.2 points)

Age-Based Urgency Calculation

Task age contributes to urgency using a linear scale based on creation date:

The urgency_age() function provides:

  • New tasks (today): 0.0 coefficient (0 points)
  • Old tasks (≥365 days): 1.0 coefficient (maximum 2.0 points)
  • Middle-aged tasks: Linear scale based on age/365 ratio

Job Blocking Urgency

Tasks with component jobs receive additional urgency when they block other jobs from completion:

Available jobs that serve as prerequisites for other jobs receive the blocking urgency bonus. This ensures that completing prerequisite tasks gets higher priority to unblock dependent work.
